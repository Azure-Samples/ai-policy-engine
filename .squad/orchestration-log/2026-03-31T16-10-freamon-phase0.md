## Notes

- Phase 0 establishes the repository pattern and CosmosDB as the durable source of truth. Redis is now exclusively a write-through cache.
- The `CachedRepository` wrapper ensures that all reads hit Redis first, with automatic fallback to CosmosDB on miss. All writes persist to Cosmos first, then update Redis.
- `RedisToCosmossMigrationService` runs on startup to detect and migrate any Redis-only data (backward compatibility for existing deployments).
- `CacheWarmingService` populates Redis from Cosmos on startup to ensure fast reads immediately after boot.
- All endpoints have been refactored to inject repository instances via DI, eliminating direct Redis dependencies.
- No breaking changes to the API surface — all endpoints retain their existing contracts.
