### Adaptive UI Logic

Billing mode computed from plan data:
- **'token'** — All plans use token billing → show only token views, hide Request Billing
- **'multiplier'** — All plans use multiplier billing → show Request Billing dashboard, hide token-only views
- **'hybrid'** — Mixed plans → show both billing models with clear labels

Pages affected:
- Navigation: "Request Billing" tab only visible for multiplier/hybrid
- RequestBilling: Empty state when no multiplier plans exist
- Export: Request billing card only visible when multiplier plans exist
- ClientDetail: Request billing section only shown for multiplier-billed plans
- RoutingPolicies: Always visible (routing useful regardless of billing mode)
