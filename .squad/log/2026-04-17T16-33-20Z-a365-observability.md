## Objective

Integrate Microsoft Agent365 Observability SDK v0.1.75-beta as an additive observability layer alongside existing OpenTelemetry and Purview DLP integrations. Support scope-based tracing for AI inference workloads.

## Implementation

### Service Architecture

```
IAgent365ObservabilityService (interface)
├── Agent365ObservabilityService (concrete impl, stub with TODO markers)
└── NoOpAgent365ObservabilityService (no-op when disabled)
```

**Methods:**
- `StartInvokeAgentScope(correlationId)` → `IDisposable?` (for agent invocation)
- `StartInferenceScope(correlationId)` → `IDisposable?` (for inference processing)

### Integration Points

1. **Dependency Injection** (`Program.cs`)
   - Extension method: `builder.AddAgent365Observability()`
   - Reads `ENABLE_A365_OBSERVABILITY_EXPORTER` env var
   - Registers either stub (enabled) or no-op (disabled)

2. **Precheck Endpoint** (`PrecheckEndpoints.cs`)
   - Both `Precheck` and `ContentCheck` handlers
   - Wrap with `using (scope = service.StartInvokeAgentScope(correlationId))`
   - Extraction of correlation ID from `X-Correlation-ID` header

3. **LogIngest Endpoint** (`LogIngestEndpoints.cs`)
   - `IngestLog` handler
   - Wrap with `using (scope = service.StartInferenceScope(correlationId))` after auth check

### SDK Maturity Notes

Current version (0.1.75-beta) lacks documented public APIs (`InvokeAgentScope.Start`, `InferenceScope.Start`, `BaggageBuilder`). Implementation is stub with TODO comments. Full implementation deferred until SDK stabilizes (0.2.x+).

## Key Design Decisions

1. **Lightweight Identity:** Use `ClientAppId` as `gen_ai.agent.id` (no Agentic User provisioning)
2. **Scope:** Precheck + LogIngest only (config CRUD excluded)
3. **Host-Level Config:** Global toggle via env var (not per-client)
4. **Local Testing:** Aspire Dashboard for OTel visualization
5. **Opt-In Exporter:** `ENABLE_A365_OBSERVABILITY_EXPORTER` (default: false)
6. **SDK Version:** v0.1.75-beta (latest NuGet; newer APIs not yet published)
